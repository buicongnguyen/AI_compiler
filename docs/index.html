WHAT IS ACTUALLY OPTIMIZED?

The answer is
not just code size.

The output must produce exactly the same tree-hash result. Within that constraint, the compiler reshapes the program to execute fewer, fuller bundles.

FIXEDProgram meaning

same indices · same values · same memory result

→
MINIMIZEDExecuted bundles

one non-debug bundle = one VM cycle

CYCLESPrimary objective

Executed cycles

The benchmark score is driven by how many non-debug VLIW bundles the VM executes. One legal bundle advances the machine by one cycle.

WORKRemove or combine

Instruction count

Dead instructions, repeated expressions, avoidable branches, and redundant address arithmetic consume scarce slots even when they are individually cheap.

MEMLoad/store slots

Memory traffic

The machine has only two load and two store slots per bundle. Keeping values in scratch and reusing proven loads reduces a frequent bottleneck.

SIMD8 lanes per vector

Lane utilization

VALU instructions operate on eight contiguous scratch words. Vectorization pays when eight independent scalar operations can share one vector opcode.

ILPFill parallel slots

Engine utilization

VLIW is only fast when independent ALU, VALU, load, store, and flow operations can occupy the same bundle without hazards.

REGHard capacity

Register pressure

Unrolling and vectorization create more live values. Those values must still fit in the 1,536-word per-core scratch file, including alignment holes.

Why the targets conflict

Unrolling exposes more parallel work but increases code size and live values. Vectorization cuts instruction count but consumes aligned 8-word registers. Scheduling for maximum engine occupancy can lengthen live ranges. The compiler is balancing a system, not maximizing one number in isolation.

THE OPTIMIZATION PLAYBOOK

Eight moves.
One objective.

Minimize cycles without changing the tree traversal and hash result.

01Parallelism

Expose the work

Loop unrolling duplicates the hot traversal body, revealing operations from several iterations that can execute together.

RESULT · fewer loop branches · larger scheduling window
02Instruction count

Delete the noise

DCE removes unused results. CSE reuses repeated expressions. Copy propagation, folding, and CFG cleanup repeatedly shrink the program as new opportunities appear.

RESULT · less work · shorter dependency chains
03Memory traffic

Promote memory

SROA turns small aggregate and table accesses into SSA values; load elimination and DSE remove redundant traffic when alias analysis proves it safe.

RESULT · loads become value reuse · dead stores vanish
04ALU operations

Make arithmetic cheaper

Strength reduction replaces repeated induction arithmetic, while MAD synthesis recognizes multiply-plus-add shapes supported directly by the vector engine.

RESULT · multiple ops collapse into one machine op
05Data parallelism

Pack eight lanes

Superword-level parallelism groups isomorphic scalar operations into the VM’s 8-lane vectors, including contiguous memory and gather-like patterns.

RESULT · up to 8 scalar values per vector instruction
06Bundle occupancy

Schedule the graph

A dependency graph decides what is ready. Delay-aware list scheduling prioritizes work that unblocks loads, fills engine slots, and can stagger independent streams.

RESULT · fewer empty slots · fewer dependency stalls
07Scratch pressure

Spend registers wisely

Live ranges are mapped into 1,536 scratch words. The default pipeline disables spilling, so allocation must fit scalars and aligned 8-word vectors.

RESULT · reuse scratch safely · avoid allocation failure
08Cycle count

Bundle for width

Independent operations share a cycle: up to 12 scalar ALU, 6 vector ALU, 2 loads, 2 stores, and 1 flow slot.

RESULT · more useful operations retired each cycle

TERMINOLOGY, DECODED

The jargon,
in plain English.

Filter the glossary by compiler level, analysis concept, optimization pass, or machine term.

01
IR

IR

Intermediate Representation

The compiler’s internal program format. It is easier to analyze and rewrite than source code or final machine bundles.

02
IR

HIR

High-level IR

The structured, SSA-oriented level where loops, conditions, pointers, assumptions, and vector intent are still visible.

03
IR

LIR

Low-level IR

A flatter representation closer to the target instruction set. Control flow and machine-like operations are explicit enough for scheduling.

04
IR

MIR

Machine IR

Scheduled machine operations grouped around engine constraints, before virtual scratch locations become final physical addresses.

05
Machine

VLIW

Very Long Instruction Word

One instruction bundle explicitly names several independent operations that different execution engines perform in the same cycle.

06
Machine

SIMD / VALU

Single Instruction, Multiple Data

One vector opcode performs the same operation across eight lanes. VALU is the VM’s vector arithmetic engine.

07
Analysis

SSA

Static Single Assignment

Each value is defined once. This makes data flow, replacement, dead-code removal, and expression reuse easier to prove.

08
Analysis

CFG

Control-Flow Graph

Basic blocks connected by possible jumps. Simplifying it removes unreachable blocks, trampolines, and redundant branches.

09
Analysis

DDG

Data-Dependency Graph

A graph whose edges say which operations must precede others. The scheduler may pack nodes together only when hazards permit.

10
Analysis

Alias analysis

Can two addresses overlap?

A proof system for memory addresses. If two pointers cannot refer to the same location, loads and stores can move or disappear safely.

11
Pass

DCE / DSE

Dead Code / Dead Store Elimination

DCE removes computations whose results are unused. DSE removes writes that cannot affect a later observable read.

12
Pass

CSE

Common Subexpression Elimination

If the same pure expression is computed twice with the same inputs, reuse the first result instead of doing the work again.

13
Pass

SROA

Scalar Replacement of Aggregates

Break a small array, table, or aggregate into independent SSA values so later passes can optimize each element directly.

14
Pass

SLSR

Straight-Line Strength Reduction

Recognize related repeated arithmetic and derive later values from earlier ones with cheaper incremental operations.

15
Pass

SLP

Superword-Level Parallelism

Find similar independent scalar statements and pack them into fixed-width vector operations without changing the source algorithm.

16
Pass

MAD

Multiply-Add

Fuse a multiply followed by an add into the target’s multiply_add operation when use counts and dependencies make it legal and profitable.

17
Machine

Register pressure

Live storage demand

How many scratch words must hold simultaneously live values. Vectors count as eight words and may require aligned contiguous space.

18
Machine

RAW hazard

Read After Write

A consumer cannot read a producer’s new value in the same bundle because all slots read the pre-bundle state. It must wait at least one bundle.
